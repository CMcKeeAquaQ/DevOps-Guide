Overview

DevOps is a simulated client production system.
It will consist of maintaining a TorQ stack and answering client queries, as well as developing new additional features.
Work as a team - work together on issues and requirements.
Main priority unless told otherwise.

Development

Software Development Life Cycle (SDLC)

  • Planning/Analysis:
    • Create issues and project boards on GitHub
    • Assign who will be taking on each issue
  • Design/Implementation/Testing:
    • Develop tasks, including unit tests
    • Use git as version control on development environment: create one branch per feature and commit all modifications (early and often!)
  • Maintenance:
    • Add or update documentation where applicable
    • Use git as version control on development environment: create one branch per feature and commit all modifications (early and often!)
  • Open a Pull Request (PR) on GitHub and let colleagues/codeowners review:
    • PRs are a place for discussion of code
    • Modify code as required by PR reviews and comments
    • Merge PR to master branch once approvals from codeowners are given
    • NB: Teammates should review each other's code
  • Carry out a release procedure:
    • Bring down environment
    • Tag release and update code in production environment
    • Start environment up
    • Run checks to ensure system stability

Operations

In terms of operations, there are several important factors:

  1. Support
  2. Monitoring
  3. Handling Bugs

Objectives

  • Handling support queries in a quick/efficient and polite manner
  • Morning monitoring checks of system, ensuring everything is up and running, etc.
    • Good to spot problems before clients
  • Fix a bug that crops up in a quick/efficient manner
    • Notify clients of error and explain system will be down temporarily while implementing fix
  • Understanding the implications of bouncing a process - feeds go down
  • Understanding implications of performing upgrades and version changes of software
  • Understanding how to manage and perform roll backs of code
  • Debugging: check logs when something is broken to see if you can find error

Environments

  • Production (Prod)
    • In the devops1 home directory
    • Data availability here is essential
    • Should only be updated with releases
    • Monitor functionality every morning
  • Development (Dev)
    • Your personal stack for developing new features
    • May be set up in your personal home directory
    • Does not need to have the same structure as Prod
