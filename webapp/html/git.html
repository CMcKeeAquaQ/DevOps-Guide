Branches

Branches are used to develop new features or changes to the repo without affecting the master/main branch.

You should NEVER commit to the master branch.
All changes should be made through other branches.

Branch names should not be named after the author, but should describe the feature or change being made.

Commit messages

Commit messages should:
  • Be descriptive of the change being made
  • Be no longer than 50 characters
  • Have the first letter capitalised
  • Use the imperative case
  • e.g. "Update CODEOWNERS to include new DevOps members"

Useful Git commands

Command Description Example
clone Clones a repository to your current directory git clone https://github.com/AquaQAnalytics/devops1-TorQ
branch Show which branch you are on and which branches are in your workspace git branch
checkout Changes which branch you are git checkout eodsummary
Can be used to create a new branch git checkout -b newbranch
pull Pulls updates for the branch you are requesting (or main if not specified) from the remote repository to your local repository git pull origin eodsummary
status Shows the status of the working directory and staging area git status
add Adds files you have changed to the staging area so they are ready for a commit git add --all
commit Commits the changes you have made and added to the staging area to your local repository git commit -m "Add end of day summary functions to rdb.q"
push Pushes any changes you have commited from your local repository to the remote (online) repository git push origin eodsummary
diff Shows changes between current working branch, and your last commit git diff
