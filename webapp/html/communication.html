Slack

The DevOps slack chat is the primary place of communication for DevOps.

Here, you can communicate with the DevOps managers, who will distribute tasks, ask questions, and check up on progress.
It is also a place where you can ask questions about anything you don't understand, or would like input on.

Standup

At 09:30 each day, there will be a standup message sent to the #devops slack channel. 

Reminder: Standup:
    · What I did yesterday
    · What I'm going to do today
    · Any blockers?

You are expected to provide an update at this time to discuss your progress, highlight any issues you've encountered, as well as including other things you are working on outside of DevOps, so we have an idea of how busy you are.

e.g. 

Yesterday: Investigated replay of logs, functionality to calculate total PnL from start of day if stack is bounced.
Today: AWS training, post rollover functionality, replay logs functionality
Blockers: None

Support Questions

In the slack chat, the managers will also act as clients, and will ask support questions that will need to be answered. The way you reply to clients is important, using a more formal writting style. 

Dear client,

Main text body

Regards,
Devops Team
  • When contacting managers, a less formal style can be used.
  • If there are any issues, or questions, communicate them here.
  • If the production stack needs restarted, managers should be informed here when data is not available, and when functionality has been restored.
